# amigo Web/PWA — Overview

Turn the terminal trainer into an **installable PWA** that runs on an Android phone
(added to the home screen, full-screen, offline) and in any browser — implementing
the mobile design `amigo - App.dc.html` (Claude Design project *Classical*).

The terminal CLI stays. Both frontends drive the **same pure engine** (`src/engine/*`).
No content or logic is rewritten — only a new Preact frontend + browser-side
persistence + a build/packaging layer are added.

## Decisions (locked with the user)

- **Delivery:** installable PWA (works in-browser *and* installs to Android; offline; no accounts). Matches the design brief's "offline, deterministic, no login" ethos.
- **UI stack:** **Preact + Vite** (+ `vite-plugin-pwa`). Preact (~4 KB) gives clean state-driven rendering for the inputs, live 663-entry vocab filtering, and screen switching. TTS / speech-to-text arrive later as framework-agnostic Web Speech APIs, so the choice doesn't constrain them.
- **CLI fate:** keep both. `src/cli/*` is untouched; `src/engine/*` stays framework-agnostic and shared.
- **We take from the design project:** the markup structure of `amigo - App.dc.html` and the `Classical` design-system `styles.css` (tokens + `.btn/.input/.tag/.card/.dialog` classes). We do **not** ship Claude Design's `_ds_bundle.js` runtime — the `.dc.html` `DCLogic`/`React.createElement` code is reimplemented in Preact.

## Architecture at a glance

```
content/*.yaml ──(build:content, Node)──► src/web/content.generated.json ──import──►┐
                                                                                     │
src/engine/*  ── pure, shared, unchanged ────────────────────────────────────►  Preact app (src/web)
  pickVocabCard / pickVerbDrill / pickSentence                                       │
  gradeVocab / gradeVerb / gradeSentence                                             │
  checkAnswer · markKnown/markUnknown · currentModule · srs                          │
                                                                                     ▼
                                             localStorage  ◄── web store (YAML text, = user.yaml)
                                             vite-plugin-pwa ── manifest + service worker (full offline precache)
```

The engine already exposes exactly the calls the frontend needs (verified against
`src/engine/trainer.ts`, `learner.ts`, `check.ts`, `srs.ts`). A **session** is the
CLI's `pick → grade → next` loop, run for 8 cards — the frontend reuses the same
functions, one card at a time (SRS state changes between picks).

## Dependencies (explicit — nothing added silently)

| Package | Type | Why |
|---|---|---|
| `preact` | runtime | UI |
| `yaml` | runtime | **already a dependency**; reused for user-state (localStorage) + Export/Import (keeps the "everything is `user.yaml`" promise) |
| `vite` | dev | dev server + bundler |
| `@preact/preset-vite` | dev | Preact JSX/HMR for Vite |
| `vite-plugin-pwa` | dev | manifest + service worker (offline install) |

No CSS framework (we ship the DS `styles.css`). No router (screen state is one enum).
Fonts are **self-hosted woff2 assets** (not an npm dep) so offline works. `tsx` and
`typescript` are already present and reused by the content build step.

## Directory layout (added)

```
index.html                     Vite entry (repo root)
vite.config.ts
scripts/build-content.ts       loadContent() → src/web/content.generated.json
src/web/
  main.tsx                     mounts <App/>
  app.tsx                      screen state machine + top-level state
  content.ts                   imports content.generated.json → typed Content
  store.ts                     localStorage ↔ UserState (YAML), export/import
  session.ts                   session runtime + CardVM normalization
  engine.ts                    thin re-export barrel of the engine calls used
  theme.ts / styles/ds.css     Classical tokens (+ dark), fonts.css (@font-face)
  screens/  Home Practice Summary Modules Settings ModuleDetail VocabEditor Focus
  components/  StatusCard TrainerRow ProgressSegments AccentKeys ModuleRow
               Checkbox Chip VocabRow SettingRow UnlockDialog
public/
  icons/  (192, 512, maskable, apple-touch)
  fonts/  (Cormorant Garamond, Lora — woff2)
```

`src/web/content.generated.json` is git-ignored and regenerated by `predev`/`prebuild`.

## Open questions / deltas from the design mock (flagged, non-blocking)

1. **Focus tags `gender_trap` / `false_friend`** shown as chips in the design are **not** in the engine `Tag` union (`gender_trap` is a boolean flag on nouns; `false_friend` isn't modeled). *Default:* the Focus screen offers the real `Tag` values (`nationality, profession, family-role, place, food, …`); a `gender_trap` filter can be added later. Confirm if you want it in the first cut.
2. **"Fokus-Sitzung starten"** — which trainer runs? The CLI applies a focus globally to whichever trainer you start next. *Default:* saving a focus returns to Home with the focus chip active; the three trainers then run focused. Confirm vs. launching a specific trainer directly from the Focus screen.
3. **Dark theme palette** — the `Classical` `styles.css` is light-only; the brief requires dark. *Default:* a first-pass dark ramp derived from the same terracotta/gold hue (see 02), to be tuned. Not a new brand color, just tone mapping.
