# 01 — Architecture & Build

## Engine reuse (no changes)

`src/engine/*` is pure and **browser-safe** — verified: zero `node:` imports, no
`fs`/`path`/`process`. The only file I/O lives in `src/cli/store.ts` (stays CLI-only).
The web app imports engine functions directly; Vite/esbuild bundles the extensionless
relative imports (`from './conjugate'`) fine. tsconfig already uses
`moduleResolution: bundler`, `module: ESNext`, `target: ES2022`.

## Content build step

The engine needs an assembled `Content` object. `src/cli/store.ts::loadContent(dir)`
already builds it from `content/*.yaml` using Node `fs` + `yaml`. Reuse it in a build
script so the browser never parses YAML or hits the filesystem:

`scripts/build-content.ts`
```ts
import { writeFileSync } from 'node:fs'
import { loadContent } from '../src/cli/store'
const content = loadContent(new URL('../content', import.meta.url).pathname)
writeFileSync(
  new URL('../src/web/content.generated.json', import.meta.url),
  JSON.stringify(content),
)
```

- Run with `tsx` (already a devDependency).
- `src/web/content.ts` does `import content from './content.generated.json'` and
  exports it typed as `Content` (`export default content as unknown as Content`).
- Bundled into the JS (≈40 KB gzip) → available offline with zero fetch.
- `content.generated.json` is git-ignored; regenerated by `predev`/`prebuild` so it's
  always fresh after content edits. It is also re-run by anyone building the app.

## Preact JSX / tsconfig

Add a web tsconfig so JSX doesn't leak into the engine/CLI build:

`src/web/tsconfig.json`
```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "jsx": "react-jsx",
    "jsxImportSource": "preact",
    "types": [],
    "lib": ["ES2022", "DOM", "DOM.Iterable"]
  },
  "include": ["./**/*.ts", "./**/*.tsx", "./content.generated.json"]
}
```
Root `tsconfig.json` stays `types:["node"]` for engine+CLI. `resolveJsonModule` is
implied by `bundler` resolution; add it explicitly if `typecheck` complains.

## Fonts (offline)

`styles.css` `@import`s Google Fonts (Cormorant Garamond, Lora). A remote `@import`
breaks offline and the PWA precache. Fix:
- Download the needed weights (Cormorant Garamond 400/600, Lora 400/600) as `woff2`
  with Latin + Latin-Extended (covers á é í ó ú ñ ü ¿ ¡) into `public/fonts/`.
- Replace the `@import` with local `@font-face` rules in `src/web/styles/fonts.css`.
- Keep the CSS var names (`--font-heading`, `--font-body`) unchanged.

## Running on the phone

Two contexts, one caveat — **service workers require a secure context** (HTTPS or
`localhost`). A phone hitting `http://<LAN-IP>:5173` is *not* secure, so the app runs
but **install/offline won't activate** in plain LAN dev.

- **Fast iteration (LAN, no install):** `npm run dev` → open `http://<PC-LAN-IP>:5173`
  on the phone (same Wi-Fi). Full app, live reload; PWA features dormant.
- **Real install + offline (recommended):** deploy the static `dist/` over HTTPS and
  open it on the phone → browser menu → *Add to Home screen* / *Install*.
  - **GitHub Pages** (repo is already `github.com:gergol/amigo`): `npm run build`,
    publish `dist/` to the `gh-pages` branch (or Actions). `base:'./'` handles the
    `/amigo/` subpath. → `https://gergol.github.io/amigo/`.
  - **Netlify/Vercel drop:** drag `dist/` — instant HTTPS URL.
  - **Local HTTPS (optional):** `vite preview` behind a self-signed cert / a
    `cloudflared`/`ngrok` tunnel if you want to install a dev build without deploying.

Document the chosen path in the README once picked.
